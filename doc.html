Installation Guide

To prepare the test environment it is enough to setup the following required libraries and packages, including dependencies mentioned below.

Mandatory Tools and Extensions

To make the workspace executable, Python and Py.test have to be installed. First of all check that you don't already have Python installed by entering python in a command line window. If you see a response from a Python interpreter it will include a version number in its initial display. If not, then:

  • python2.X
    Installation:
    $sudo apt-get install python2.X
  • py.test
    Installation:
    $sudo apt-get install python-pip
    $sudo pip install --upgrade pip
    $sudo pip install --upgrade pytest

As all the data templates use yaml format, the yaml library for Python is required:

  • yaml
    Installation:
    $sudo pip install setuptools --upgrade
    $sudo pip install pyyaml

To enable ssh or telnet connection to the remote devices, Python implementation of the SSHv2 protocol is required:

  • paramiko
    Installation:
    $sudo pip install paramiko

By default py.net.test uses Ostinato traffic generator (server), which can be installed on the local or remote host (for Ubuntu 13.10 and later):

  • Ostinato (TODO)
    Installation:
    $sudo apt-get install ostinato

    For other environment see Ostinato wiki.
    NB: py.net.test uses its own python binding package (client), which has to connect to the Ostinato server.
  • Ostinato Client Dependencies
    Installation:
    $sudo apt-get install python-protobuf

To check the installed Python packages, run:

$pip freeze

Optional Tools and Extensions

To generate doxygen documentation on the fly, install Doxygen (v. 1.8.8) and Doxypy tools:

  • Doxygen
    Installation: Sources and manual
  • doxypy
    Installation:
    $sudo apt-get install doxypy

To use the py.net.test Topology Builder tool, install PySide - Python bindings for the Qt cross-platform application, and its dependencies:

  • PySide Dependencies
    Installation:
    $sudo apt-get install build-essential git cmake libqt4-dev libphonon-dev python2.7-dev libxml2-dev libxslt1-dev qtmobility-dev
  • PySide
    Installation:
    $sudo apt-get install python-pyside

    NB: sometimes PySide is installed to the wrong directory, and it is necessary to move the package to /usr/bin/ manually:
    $ls /usr/local/bin
    $sudo cp /usr/local/bin/pyside* /usr/bin/

To have visualized and flexible test reports, it is necessary to install the Allure plugin:

  • Allure
    Installation:
    $sudo pip install -i https://pypi.binstar.org/pypi/simple pytest-allure-adaptor
    $sudo pip install pytest-allure-adaptor
    $sudo apt-add-repository ppa:yandex-qatools/allure-framework
    $sudo apt-get install allure-cli

For py.net.test developers it is strongly recommended to use the Unittests coverage plugin and PEP8 code reviser:

  • cov
    Installation:
    $sudo pip install pytest-cov
  • git-pre-commit-hook
    Installation:
    $sudo pip install frosted
    $sudo pip install flake8
    $sudo git-pre-commit-hook install --plugin flake8 --plugin frosted --plugin yaml

For more information on the Py.test plugins, see List of Third-Party Plugins
